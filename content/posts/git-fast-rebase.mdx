# Git 快速重新链接

在使用 Git 和 GitHub 的过程中，你可能会遇到这样的情况：**明明之前还能正常推送代码，过了一段时间后，VS Code 突然无法上传到仓库了**。或者提示 `Permission denied (publickey)`。

别慌，这通常只是 SSH 密钥或远程仓库 URL 的问题。本文将带你用最少的步骤，快速恢复 Git 连接。

## 前置：配置用户信息

在开始之前，确保你的 Git 全局用户信息已正确设置（与 GitHub 账号一致）：

```bash
git config --global user.name "你的GitHub用户名"
git config --global user.email "你的GitHub绑定邮箱@example.com"
```

## 场景一：全新项目克隆

如果你要从 GitHub 克隆一个仓库到本地：

1. 在 VS Code 中按 `Ctrl+Shift+P` 打开命令面板。
2. 输入 `Git: Clone` 并选择。
3. 选择 `Clone from GitHub`，然后选择仓库。
4. 或者直接输入 SSH URL：`git@github.com:用户名/仓库名.git`。

## 场景二：现有项目远程地址是 HTTPS -> 改为 SSH

很多推送失败的原因是远程仓库使用了 HTTPS 协议（每次都要输密码，且容易受系统凭据管理器影响）。建议改用 SSH。

在项目终端中检查当前远程 URL：

```bash
git remote -v
```

如果看到 `https://github.com/...`，请改为 SSH：

```bash
git remote set-url origin git@github.com:用户名/仓库名.git
```

例如：
```bash
git remote set-url origin git@github.com:JiangTaiBai1/vegitable.git
```

## 场景三：SSH 连接失败 / 密钥问题

如果执行 `git push` 后出现类似错误：
> `Permission denied (publickey)`
> `Could not read from remote repository`

### 首先

### 3.1 测试 SSH 连接

```bash
ssh -T git@github.com
```

- **成功** 会显示：
  ```
  Hi 用户名! You've successfully authenticated, but GitHub does not provide shell access.
  ```
- **失败** 会提示 `Failed to add the host to the list of known hosts` 或 `Permission denied`。

### 3.2 启动 ssh-agent 并添加密钥

```bash
# 启动 ssh-agent（后台进程）
eval $(ssh-agent -s)

# 添加私钥（默认路径）
ssh-add ~/.ssh/id_ed25519
```

如果提示 `Could not open a connection to your authentication agent`，请先执行 `eval $(ssh-agent -s)`。

### 3.3 检查并生成新密钥（如果没有）

查看是否已有密钥：
```bash
ls ~/.ssh/
```

如果没有 `id_ed25519` 或 `id_rsa`，则生成新密钥（邮箱填写你的 GitHub 邮箱）：

```bash
ssh-keygen -t ed25519 -C "你的邮箱@example.com"
```

一路回车使用默认路径和空密码即可。

### 3.4 显示公钥并添加到 GitHub

```bash
cat ~/.ssh/id_ed25519.pub
```

复制输出的全部内容（以 `ssh-ed25519` 开头，以你的邮箱结尾）。

然后：
1. 登录 GitHub → 右上角头像 → **Settings**。
2. 左侧菜单 → **SSH and GPG keys**。
3. 点击 **New SSH key**，粘贴公钥，保存。

![SSH 保存github的ssh秘钥管理中](/images/posts/GitHub-ssh-key-save.png)

### 3.5 再次测试连接

```bash
ssh -T git@github.com
```

现在应该看到成功的欢迎信息。

## 常见问题：VS Code 无法识别 Git

如果你的 VS Code 源代码管理面板中提示“未找到 Git”，请按以下步骤解决：

1. **检查 Git 是否在系统 PATH 中**  
   打开终端，输入 `git --version`。如果提示“不是内部或外部命令”，请先安装 Git（[git-scm.com](https://git-scm.com/)）。

2. **在 VS Code 设置中手动指定 git.path**  
   - 按 `Ctrl+,` 打开设置。
   - 搜索 `git.path`。
   - 点击 `在 settings.json 中编辑`，添加：
     ```json
     "git.path": "C:\\Program Files\\Git\\bin\\git.exe"
     ```
     （根据你的实际安装路径修改）

3. **重启 VS Code**。

## 密钥权限修复（仅 Windows + WSL / Git Bash 用户）

某些情况下 SSH 会因私钥权限过于开放而拒绝使用。在 Git Bash 或 WSL 中执行：

```bash
chmod 700 ~/.ssh
chmod 600 ~/.ssh/id_ed25519
chmod 644 ~/.ssh/id_ed25519.pub
```

## 完整验证清单

做完上述步骤后，按顺序验证一下：

- ✅ 生成了 SSH 密钥（或已有）
- ✅ 公钥已添加到 GitHub
- ✅ `ssh -T git@github.com` 成功
- ✅ `git config --global user.name/email` 已设置
- ✅ 远程仓库 URL 为 SSH 格式
- ✅ VS Code 中能正常推送/拉取

## 最后：测试推送

在项目文件夹中做一次提交并推送：

```bash
git add .
git commit -m "修复 SSH 连接"
git push -u origin main
```

如果一切正常，你的代码就顺利推送到 GitHub 了。

---

**总结**：大多数 Git 连接问题都源于 SSH 密钥未正确加载或远程 URL 使用了 HTTPS。按本文三步走——**生成密钥 → 添加到 GitHub → 改用 SSH 地址**——即可快速解决。
